# project-unicorn
=БАЗЫ ДАННЫХ=

БД - набор данных, хранящихся в структурированном виде
- каждый предмет имеет своё место и маркировку для быстрого поиска
- хранятся и обрабатываются в !элект.виде в компьютерной системе!

Система управления БД (СУБД) - система, позволяющая создавать БД и манипулировать сведениями из них

Взаимодействие с БД:
User хочет получить данные из БД -> формирует запрос на языке SQL -> СУБД принимает запрос и обращается к БД для поиска нужной ин-ции -> СУБД возвращает найденные данные user

-ТИПЫ БД-
1) Реляционные БД - данные организованы в виде связанных таблиц, для работы используют язык SQL (декларативный язык программирования, т.е. описывается ожидаемый рез-тат, а не способ его получения), с помощью него можно создавать/изменять объекты БД; запрашивать данные, фильтруя/сортируя/объединяя их; добавлять/обновлять/удалять записи; выполнять операции агрегации(sum, count, min, max, etc) и т.д.

a) PostgreSQL - бесплатная СУБД с открытым исходным кодом, отлично работает на linux и macos; требует следования стандартам SQL

б) MsSQL - СУБД от Microsoft, коммерческий продукт, родная платформа - windows; процедурный язык с большим кол-вом встроенных функций; простота написания сложных запросов

в) MySQL - бесплатная СУБД с открытым исходным кодом; версия языка в MYSQL близка к SQL; часто применяется в веб-разработке. MariaDB - ответвление от mysql, более новая и продвинутая версия

2) Нереляционные БД (NoSQL) - работают с большими объёмами неструктурированных данных (схема данных варьируется от записи к записи или часто меняется со временем, сложные взаимосвязи между объектами, которые невозможно эффективно моделировать в табличном формате); отказ от табличной структуры, как в РБД, в пользу альт. моделей хранения:

- Документоориентированные: хранят данные в виде документов, которые могут содержать вложенные объекты и массивы: JSON-документы - полуструктурированные данные, которые хранятся не в таблицах, а в «документах». Каждый документ — это самодостаточная единица, содержащая все данные об объекте;
- Ключ-значение: Ключ (Key) - уникальный идентификатор записи. Это может быть строка, путь к файлу или хэш. По ключу происходит поиск. Значение (Value) - сами данные. Для базы данных это часто «чёрный ящик» - хранилище не знает, лежит там число, JSON-объект или картинка;
- Колоночные: табличный формат данных, но с динамическим количеством столбцов. Суть: данные хранятся не по строкам (как в SQL), а по колонкам. Это позволяет сжимать данные и невероятно быстро агрегировать их (считать суммы, средние значения) по миллиардам строк.
- Графовые: узлы (объекты) и ребра (связи), запросы перемещаются по графу, следуя соединениям, а не вычисляя их во время запроса.

а) MongoDB - СУБД для работы с документоориентированными БД. Базы состоят из коллекций (группа документов, похожая на таблицу, но без фиксированной схемы) и документов (основная единица хранения, представленная в виде BSON-объекта (бинарный JSON). Документ - набор полей, каждое из которых представляет собой пару «ключ → значение»);

б) ClickHouse - СУБД колоночного типа, предназначенная для аналитической обработки больших объёмов данных;

в) Redis - СУБД, которая хранит данные в формате «ключ-значение». Хранит данные в оперативной памяти, работает быстрее, чем системы, которые используют жёсткий диск. Высокая скорость: для кэширования (ускорение работы приложений путём временного хранения часто используемых данных в быстром доступе), производительность веб-приложений.

-ПРОГРАММЫ ДЛЯ АНАЛИЗА-
1) Клиент базы (Database Clients) - программы, основная задача которых - подключиться к БД, выполнить запрос и показать результат. Запросы пишутся вручную, можно просматривать и фильтровать данные, экспортировать результаты в CSV/JSON, визуально исследовать схему данных;

2) Инструменты для написания запросов (IDE / Query Builders) - для удобной разработки, помогают писать код быстрее и с меньшим количеством ошибок;

3) Инструменты аналитики и бизнес-разведки (BI, Data Visualization) - не требуют от пользователя глубокого знания языка запросов, часто работают через SQL или специальные коннекторы, но основная ценность — в визуальном исследовании и представлении данных;
